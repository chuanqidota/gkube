K8s 多集群管理开源项目调研

主流方案对比 — 为 gkube 设计提供参考

🏠 Kuboard

定位：国内团队开发的轻量级 K8s 管理界面

架构：Go + Vue.js 3，前后端分离 Web 应用

亮点：中文友好、UI 直观、部署简单

多集群：v3+ 支持，按环境分组

✅ 适合国内中小团队快速上手

🏢 Rancher

定位：SUSE 旗下企业级多集群管理平台

架构：Go + React，中心化管理平面

亮点：集群生命周期管理、GitOps、安全合规

多集群：业界领先，支持数百个集群

⚠️ 系统较重，资源占用大

🌐 KubeSphere

定位：青云开源的云原生应用管理平台

架构：Go + React，微内核+扩展(LuBan)

亮点：功能全面(监控/日志/DevOps/服务网格)

多集群：主集群+成员集群 Agent 模式

⚠️ 复杂度高，学习曲线陡峭

💡 Headlamp

定位：Microsoft 支持的轻量级 K8s Web UI

架构：Go + React/TypeScript，插件系统

亮点：轻量、可扩展、Apache 2.0

多集群：基础支持

✅ 轻量级可扩展，插件架构优秀
