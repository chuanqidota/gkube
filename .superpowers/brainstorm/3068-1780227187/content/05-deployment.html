设计第五部分：部署方案

独立部署 + 集群内部署 双模式

🐳 部署模式一：独立部署

Docker Compose 编排
# docker-compose.yaml
version: '3.8'
services:
  backend:
    build: ./backend
    ports:
      - "8080:8080"
    depends_on:
      - mysql
      - redis
    environment:
      - DB_HOST=mysql
      - REDIS_ADDR=redis:6379

  frontend:
    build: ./frontend
    ports:
      - "80:80"
    depends_on:
      - backend

  mysql:
    image: mysql:8.0
    volumes:
      - mysql-data:/var/lib/mysql
    environment:
      - MYSQL_ROOT_PASSWORD=xxx
      - MYSQL_DATABASE=gkube

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"

☸️ 部署模式二：集群内部署

Helm Chart 部署
# 使用 Helm Chart 部署到 K8s 集群
helm install gkube ./deploy/helm/gkube \
  --namespace gkube-system \
  --create-namespace \
  --set backend.replicas=2 \
  --set frontend.replicas=2 \
  --set mysql.enabled=true \
  --set redis.enabled=true

# 或使用外部数据库
helm install gkube ./deploy/helm/gkube \
  --set mysql.enabled=false \
  --set externalMySQL.host=your-mysql-host \
  --set externalMySQL.password=your-password

📋 Makefile 常用命令

开发与构建命令
# 后端开发
make backend-dev        # 热重载开发
make backend-build      # 编译二进制
make backend-test       # 运行测试
make migrate            # 数据库迁移

# 前端开发
make frontend-dev       # Vite 开发服务器
make frontend-build     # 生产构建

# Docker
make docker-build       # 构建所有镜像
make docker-up          # 启动开发环境
make docker-down        # 停止开发环境

# 生产部署
make build              # 构建生产版本
make deploy             # 部署到 K8s

🔒 安全设计

后端安全
  • 🔒 kubeconfig AES-256 加密存储
  • 🔒 密码 bcrypt 加盐哈希
  • 🔒 JWT Token 过期时间 2 小时
  • 🔒 Refresh Token 过期时间 7 天
  • 🔒 RBAC 权限校验中间件
  • 🔒 请求频率限制（防暴力破解）
  • 🔒 操作审计日志
前端安全
  • 🔒 Token 存储在 localStorage
  • 🔒 Axios 拦截器自动附加 Token
  • 🔒 401 自动跳转登录页
  • 🔒 路由守卫防止未授权访问
  • 🔒 敏感操作二次确认弹窗
  • 🔒 XSS 防护（Vue 模板转义）
  • 🔒 CSRF Token（可选）
