设计第二部分：数据模型

核心实体设计 — 认证、集群、RBAC

👤 用户与角色模型

User / Role / Permission
// 用户表
User {
  ID            uint       `gorm:"primaryKey"`
  Username      string     `gorm:"unique;size:50"`
  PasswordHash  string     `gorm:"size:255"`         // bcrypt 加密
  Email         string     `gorm:"size:100"`
  DisplayName   string     `gorm:"size:100"`
  Status        int        `gorm:"default:1"`        // 1=启用 0=禁用
  Roles         []Role     `gorm:"many2many:user_roles;"`
  CreatedAt     time.Time
  UpdatedAt     time.Time
}

// 角色表
Role {
  ID            uint          `gorm:"primaryKey"`
  Name          string        `gorm:"unique;size:50"`   // admin, viewer, operator
  Description   string        `gorm:"size:200"`
  Permissions   []Permission  `gorm:"many2many:role_permissions;"`
  CreatedAt     time.Time
  UpdatedAt     time.Time
}

// 权限表
Permission {
  ID            uint   `gorm:"primaryKey"`
  Resource      string `gorm:"size:100"`   // cluster, pod, deployment, ...
  Action        string `gorm:"size:50"`    // create, read, update, delete, exec
  Description   string `gorm:"size:200"`
}

🖥️ 集群模型（增强版）

K8SCluster — 在现有模型上扩展
K8SCluster {
  ID              uint       `gorm:"primaryKey"`
  ClusterName     string     `gorm:"unique;size:100"`    // 集群名称
  DisplayName     string     `gorm:"size:100"`           // 显示名称
  Description     string     `gorm:"size:500"`           // 描述
  KubeConfig      string     `gorm:"size:12800"`         // kubeconfig（AES加密存储）
  Status          string     `gorm:"size:20"`            // online/offline/unknown
  ClusterVersion  string     `gorm:"size:100"`           // K8s 版本
  NodeCount       int                                   // 节点数量
  LastHealthCheck time.Time                             // 最后健康检查时间
  Labels          string     `gorm:"size:500"`           // JSON 标签 {"env":"prod","region":"cn-east"}
  CreatedAt       time.Time
  UpdatedAt       time.Time
}

// 集群标签用于分类筛选，如：
// {"environment": "production", "region": "cn-east", "provider": "aliyun"}

🔐 RBAC 权限模型设计

三级权限控制
集群级别
指定用户可访问哪些集群
cluster:prod-a → read
命名空间级别
指定用户可访问哪些命名空间
cluster:prod-a/ns:default → read
资源级别
指定用户可操作哪些资源
pod → create,read,delete
预设角色： super_admin（全部权限） | cluster_admin（指定集群全部权限） | developer（指定命名空间读写） | viewer（只读）
